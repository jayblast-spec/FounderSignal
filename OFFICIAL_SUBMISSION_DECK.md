# FounderSignal Submission Deck

## Slide 1 - Title

FounderSignal: The Autonomous Idea-to-Execution Compiler

Turns founder instinct into Codex-ready system blueprints.

## Slide 2 - Problem

Founders do not fail only because they lack code. They fail because raw intent turns into vague backlog, unclear buyer assumptions, unsafe data design, and untestable product scope.

## Slide 3 - Solution

FounderSignal compiles startup intent into a deterministic execution packet:

- `spec.md`
- `schema.sql`
- `tasks.txt`
- `manifest.json`
- GitHub loop packet
- Vault handoff commit

## Slide 4 - Agentic Workflow

1. Select one-click demo.
2. Run agent confrontation.
3. Compile artifacts.
4. Inject practitioner correction.
5. Recompile with regression check.
6. Commit corrected artifact state to Vault handoff.

## Slide 5 - Self-Improving Loop

The correction loop does not append notes. It mutates artifacts in place.

Demo phrase:

`Must support completely localized data isolation rules.`

Expected result:

- System Updated: Regression check passed
- `schema.sql` includes RLS
- `tasks.txt` includes verification checkpoints
- `manifest.json` updates as a Codex Skill payload

## Slide 6 - Direct-To-IDE Output

FounderSignal generates machine-parsable files Codex can use directly:

- Functional spec
- Supabase schema
- Atomic task checklist
- Skill manifest
- Agent instructions

## Slide 7 - Build Week Fit

- **Technical Implementation:** multi-step endpoints, compiler guardrails, diff engine, Vault commit hash.
- **Design:** one-click demos, tabbed ArtifactViewer, prominent Copy Code for Codex flow.
- **Impact:** compresses strategy-to-repository planning from hours to minutes.
